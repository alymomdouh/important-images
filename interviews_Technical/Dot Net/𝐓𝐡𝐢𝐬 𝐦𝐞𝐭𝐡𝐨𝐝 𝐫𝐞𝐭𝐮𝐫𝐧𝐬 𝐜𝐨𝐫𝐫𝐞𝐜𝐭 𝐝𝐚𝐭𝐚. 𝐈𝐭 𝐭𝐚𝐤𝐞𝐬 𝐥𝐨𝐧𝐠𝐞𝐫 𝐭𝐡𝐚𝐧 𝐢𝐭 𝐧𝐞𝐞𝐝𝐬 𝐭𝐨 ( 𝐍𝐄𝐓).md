Three independent calls.

Three awaits.

No errors.

No warnings.

But they run one after another, each waiting for the previous to finish before starting. The total time is the sum of all three.

They could all run at the same time.

# ❌ المشكلة في الكود الحالي

```csharp
var orders = await _orderService.GetOrdersAsync(userId);
var invoices = await _invoiceService.GetInvoicesAsync(userId);
var notifications = await _notificationService.GetUnreadAsync(userId);
```

👉 هنا التنفيذ **Sequential**

يعني:

```text
Total Time = orders + invoices + notifications
```

---

# ✅ أفضل حل (الـ Standard)

## ✔ استخدم `Task.WhenAll` (Parallel Execution)

```csharp
public async Task<DashboardData> GetDashboardAsync(int userId)
{
    var ordersTask = _orderService.GetOrdersAsync(userId);
    var invoicesTask = _invoiceService.GetInvoicesAsync(userId);
    var notificationsTask = _notificationService.GetUnreadAsync(userId);

    await Task.WhenAll(ordersTask, invoicesTask, notificationsTask);

    return new DashboardData
    {
        Orders = await ordersTask,
        Invoices = await invoicesTask,
        Notifications = await notificationsTask
    };
}
```

---

# 🔥 النتيجة

```text
Total Time ≈ max(orders, invoices, notifications)
```

✔ أسرع بفرق كبير

# ⚠️ أهم نقطة (الناس بتغلط فيها)

## ❌ مش دايمًا Task.WhenAll صح

### 💣 المشكلة:

لو التلات services بيستخدموا نفس **DbContext (EF Core)**

👉 هيطلع Exception:

```
A second operation was started on this context...
```

# ✅ الحل الصح مع EF Core

## ✔ استخدم `IDbContextFactory`

```csharp
public class OrderService
{
    private readonly IDbContextFactory<AppDbContext> _factory;

    public OrderService(IDbContextFactory<AppDbContext> factory)
    {
        _factory = factory;
    }

    public async Task<List<Order>> GetOrdersAsync(int userId)
    {
        using var db = _factory.CreateDbContext();
        return await db.Orders.Where(x => x.UserId == userId).ToListAsync();
    }
}
```

✔ كل Task له DbContext خاص
✔ مفيش Thread-safety issues

## 2) ✔ Partial Success (أفضل للـ Dashboard)

```csharp
var ordersTask = SafeExecute(() => _orderService.GetOrdersAsync(userId));
var invoicesTask = SafeExecute(() => _invoiceService.GetInvoicesAsync(userId));
var notificationsTask = SafeExecute(() => _notificationService.GetUnreadAsync(userId));

await Task.WhenAll(ordersTask, invoicesTask, notificationsTask);

return new DashboardData
{
    Orders = ordersTask.Result,
    Invoices = invoicesTask.Result,
    Notifications = notificationsTask.Result
};
```

```csharp
private async Task<T> SafeExecute<T>(Func<Task<T>> action)
{
    try
    {
        return await action();
    }
    catch
    {
        return default;
    }
}
```

✔ لو Service وقع → باقي الداتا تشتغل

# 🧠 أولًا: التعريف السريع

## 🔷 Concurrency (التزامن)

> تنفيذ أكتر من Task **في نفس الوقت بشكل منطقي** (مش شرط فعلي)

✔ النظام بيبدّل بينهم (switching)
✔ ممكن يكون Thread واحد

---

## 🔶 Parallelism (التوازي)

> تنفيذ أكتر من Task **في نفس اللحظة فعليًا**

✔ محتاج أكتر من CPU Core
✔ كل Task على Thread مختلف

---

# 🎯 الفرق في جملة واحدة

> **Concurrency = إدارة وقت**
> **Parallelism = استخدام موارد (CPU)**

---

# 🧪 مثال بسيط

## 🎭 Concurrency (Single Thread)

```text
Task A → شوية شغل → wait
Task B → شوية شغل → wait
Task A يكمل
```

✔ نفس الـ Thread
✔ switching بينهم

---

## ⚡ Parallelism (Multi Thread)

```text
Core 1 → Task A
Core 2 → Task B
```

✔ الاتنين شغالين مع بعض فعليًا

---

# 💻 في .NET

## 🔷 Concurrency باستخدام async/await

```csharp
var task1 = GetDataFromApi();
var task2 = GetDataFromApi();

await Task.WhenAll(task1, task2);
```

✔ مناسب لـ:

* I/O (API - DB - File)

👉 هنا مش شرط يبقى Parallel فعلي
لكن Non-blocking

---

## 🔶 Parallelism باستخدام Parallel

```csharp
Parallel.For(0, 1000, i =>
{
    DoCpuWork(i);
});
```

✔ مناسب لـ:

* CPU-intensive tasks

---

# ⚠️ الفرق المهم جدًا (اللي بيوقع ناس كتير)

## ❌ async ≠ Parallel

```csharp
await Task.Delay(1000);
```

✔ ده async
❌ مش Parallel

---

# 🧠 مثال من شغلك (Dashboard)

## ❌ Sequential

```csharp
await GetOrders();
await GetInvoices();
```

---

## ✔ Concurrency

```csharp
var t1 = GetOrders();
var t2 = GetInvoices();

await Task.WhenAll(t1, t2);
```

👉 ده **Concurrency (I/O)**

---

## ✔ Parallelism (لو CPU)

```csharp
Parallel.Invoke(
    () => CalculateReport1(),
    () => CalculateReport2()
);
```

# ⚖️ مقارنة سريعة

|          | Concurrency            | Parallelism             |
| -------- | ---------------------- | ----------------------- |
| التنفيذ  | logically simultaneous | physically simultaneous |
| threads  | ممكن 1                 | multiple                |
| الهدف    | responsiveness         | performance             |
| مناسب لـ | I/O                    | CPU                     |
| مثال     | async/await            | Parallel.For            |

# 🚀 الخلاصة الذهبية

> 🔥 Concurrency = تشغيل tasks بدون انتظار
> 🔥 Parallelism = تشغيل tasks في نفس اللحظة

# 🎯 Interview Answer (جاهز)

> Concurrency is about handling multiple tasks efficiently by interleaving execution, often using async/await for I/O-bound work.
> Parallelism is about executing multiple tasks simultaneously using multiple threads/cores, typically for CPU-bound operations.
