##  Excited to share something I’ve been working on 

I’ve just released EfCoreKit, a .NET library designed to eliminate repetitive boilerplate in EF Core so you can focus on what really matters: building features.


As developers, we often rewrite the same things in every project: pagination, soft deletes, audit tracking, repositories, and more. It gets tiring and slows us down.


 With EfCoreKit, you install the package, call a method, and all of that is handled for you.
 

 What you get out of the box:
 
 ♻️ Soft Delete with global query filters
 
 ♻️ Audit Trail (CreatedAt, UpdatedAt, etc.)
 
 ♻️ Pagination (offset & keyset)
 
 ♻️ Repository & Unit of Work
 
 ♻️ Specification Pattern
 
 ♻️ Dynamic Filters
 
 ♻️ Query Helpers & DbContext utilities
 

 Design goals:
 
No lock-in works with standard EF Core

Fully opt-in features

Clean, maintainable architecture

No more rewriting the same pagination logic or plumbing code again and again.

 Install:
 
dotnet add package EfCoreKit
